Chapter 7: Software Engineering

TERM DEFINITION
Association for Computing Machinery (ACM)
  • purpose of ACM- dedicated to advancing the arts, science, and information technology
  • benefits of ACM membership- access to digital library, online books and courses
  • student ACM membership- help to find job opportunities/ interet groups
Institute of Electrical and Electronics Engineers (IEEE)
  • purpose of IEEE- improving the state of software engineering
  • benefits of IEEE membership- access IEEE resources
  • student IEEE membership- get resources/ opportunities you need to stay on top of technology
Software Life Cycle: Development Phase
  • Requirements analysis- specifies what services the system will provide
  • Design- where requirements analysis provides a description of the software product
  • Implementation- involves the actual writing of programs, data files
  • Testing- process of debugging programs and confirming compatibility
Modular Implementation determine the underlying goal during the initial design process
Unified Modeling Language (UML)
  • use case- a modern collection of tools developed with the object-oriented paradigm in mind
  • use case diagram- interactions between the system and its users
Data Relationships:
  • one-to-one
  • one-to-many
  • many-to-many
Design Pattern predeveloped model for solving a recurring problem in software design
software quality assurance (SQA)
  • glass box testing- where the software tester is aware of the structure and uses this knowledge when designing the test
  • black box testing- refers to tests that do not rely on the knowledge of the software's composition
  • beta testing- in which a preliminary version of the software is given to learn how the software performs
  • alpha testing- similar to beta testing performed at the developer's site
Documentation
  • user documentation- used to explain the features of software and how to use them
  • system documentation- used to describe the software's internal composition for maintenance
  • technical documentation- used to describe how a software system should be installed/serviced
Intellectual property law
  • software license- where the rights of the developer are expressed in legal terms
  • software disclaimer- used to protect software developer's against liability
  • best defenses against software liability claims- to apply sound software engineering principles during development
