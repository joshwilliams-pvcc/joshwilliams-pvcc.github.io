Git/Github terms:

  1. repository- a location where data is stored and managed
  2. commit- similar to save button
  3. fork- a copy of a repository
  4. push- updates Github with local commits
  5. pull requests- used for merging code
  6. workflows- automated process to run multiple jobs
  7. issues- used to track your work
  8. raw button- blank html file
  9. blame button- determines who made changes to a file
