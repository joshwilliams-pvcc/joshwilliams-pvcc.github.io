Chapter 5: Programming Algorithms & Software Testing

Phases of computer program development:

  • Phase 1: Understand the problem
  • Phase 2: Design an algorithm to solve the problem
  • Phase 3: Write the program code in appropriate language based on the algorithm
  • Phase 4: Enter the program and save
  • Phase 5: Execute the program
  • Phase 6: Test the program, find errors and debug, then retest
  • Phase 7: Document and evaluate the program

Programming structures:

  • Sequential structure: Top-down execution
  • Decision structures: if/then/else
  • Iterative (Looping):
  • while loop: continue as long something is true
  • repeat/until loop: continue looping until something is true
  • for loop: loop a given number of times
Recursive structures:
Repeat a set of instructions as a subtask of itself
Repitition continues until:
degeneraative case, which is a termination condition, is met
(degenerative case is also called a "base case")

    Software testing and verification

    Kinds of tests: (find the definitions for these here: https://www.ibm.com/topics/software-testing)
    1. Acceptance testing: Verifying whether the whole system works as intended.
    2. Integration testing: Ensuring that software components or functions operate together
    3. Unit testing: Validating that each software unit performs as expected. A unit is the smallest testable component of an application
    4. Functional testing:Checking functions by emulating business scenarios, based on functional requirements
    5. Performance testing: Testing how the software performs under different workloads
    6. Regression testing: Checking whether new features break or degrade functionality
    7. Stress testing: Testing how much strain the system can take before it fails
    8. Usability testing:Validating how well a customer can use a system or web application to complete a task

    Software testing is done to detect:

    • Arctitectural flaws
    • Poor design decisions
    • Invalid or incorrect functionality
    • Security vulnerabilities
    • Scalability issues

    Testing best practices:

    (explain these--see the IBM web site for the information)
    • Continuous testing: Project teams test each build as it becomes available
    • Configuration management: Organizations centrally maintain test assets and track what software builds to test
    • Service virtualization: simulates the services and systems that are missing or not yet completed, enabling teams to reduce dependencies and test sooner
    • Defect or "bug tracking": Monitoring defects is important to both testing and development teams for measuring and improving quality
    • Metrics and reporting: Reporting and analytics enable team members to share status, goals and test results
